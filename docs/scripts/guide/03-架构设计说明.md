## 1. 架构概览

Lxy-Vue-Admin 采用现代化的前端架构设计，基于 **Monorepo** 架构管理多个包，使用 **分层架构** 组织代码，实现了高内聚、低耦合的模块化设计。

### 架构特点

- **Monorepo 架构**：使用 pnpm workspace + Turbo 管理多包项目
- **分层架构**：视图层、组件层、服务层、工具层、基础层清晰分离
- **模块化设计**：功能模块独立封装，支持按需加载
- **类型安全**：完整的 TypeScript 类型定义
- **响应式设计**：基于 Vue 3 Composition API 的响应式架构

## 3. 核心模块设计

### 3.1 认证授权模块

**功能职责**：

- 用户登录认证
- Token 管理（Access Token / Refresh Token）
- 用户信息获取
- 登出处理

**核心文件**：

| 文件 | 说明 |
|------|------|
| `store/auth.ts` | 认证状态管理 |
| `api/core/auth.ts` | 认证 API 接口 |
| `views/_core/authentication/` | 登录页面组件 |

**认证流程**：

```
用户输入账号密码
       ↓
前端 RSA 加密
       ↓
发送登录请求
       ↓
后端验证并返回 Token
       ↓
存储 Token 到 AccessStore
       ↓
获取用户信息
       ↓
生成动态路由
       ↓
跳转到首页
```

**核心代码**：

```typescript
// store/auth.ts
export const useAuthStore = defineStore('auth', () => {
  const loginLoading = ref(false);

  async function authLogin(params: LoginAndRegisterParams) {
    loginLoading.value = true;
    try {
      // 1. 调用登录接口
      const { access_token } = await loginApi(params);
      
      // 2. 存储 Token
      accessStore.setAccessToken(access_token);
      
      // 3. 获取用户信息
      const userInfo = await fetchUserInfo();
      userStore.setUserInfo(userInfo);
      
      // 4. 设置权限
      accessStore.setAccessCodes(userInfo.permissions);
      
      // 5. 跳转首页
      await router.push(preferences.app.defaultHomePath);
    } finally {
      loginLoading.value = false;
    }
  }

  return { authLogin, logout, fetchUserInfo, loginLoading };
});
```

### 3.2 路由权限模块

**功能职责**：

- 路由守卫配置
- 动态路由生成
- 权限菜单过滤
- 路由跳转控制

**核心文件**：

| 文件 | 说明 |
|------|------|
| `router/guard.ts` | 路由守卫 |
| `router/access.ts` | 路由权限处理 |
| `router/routes/` | 路由定义 |

**路由分类**：

| 类型 | 说明 | 示例 |
|------|------|------|
| 核心路由 | 不需要权限验证 | 登录页、404、403 |
| 动态路由 | 从后端动态获取 | 系统管理、工作流 |

**路由守卫流程**：

```typescript
// router/guard.ts
router.beforeEach(async (to, from) => {
  // 1. 检查是否为核心路由
  if (coreRouteNames.includes(to.name as string)) {
    return true;
  }

  // 2. 检查 Token
  if (!accessStore.accessToken) {
    return { path: LOGIN_PATH, query: { redirect: to.fullPath } };
  }

  // 3. 检查是否已生成动态路由
  if (accessStore.isAccessChecked) {
    return true;
  }

  // 4. 生成动态路由
  const routeInfo = await generateAccess({ router, routes: accessRoutes });
  accessStore.setAccessMenus(routeInfo.accessibleMenus);
  accessStore.setAccessRoutes(routeInfo.accessibleRoutes);
  accessStore.setIsAccessChecked(true);

  return redirectPath;
});
```

**后端路由转换**：

```typescript
// router/access.ts
function backMenuToVbenMenu(menuList: Menu[]): RouteRecordStringComponent[] {
  return menuList.map((menu) => {
    // 处理外链
    if (/^https?:\/\//.test(menu.path)) {
      menu.component = 'Link';
    }
    
    // 处理内嵌 iframe
    if (menu.meta?.link && menu.component === 'InnerLink') {
      menu.component = 'IFrameView';
    }

    return {
      component: menu.component,
      meta: {
        hideInMenu: menu.hidden,
        icon: menu.meta?.icon,
        keepAlive: !menu.meta?.noCache,
        title: menu.meta?.title,
      },
      name: menu.name,
      path: menu.path,
    };
  });
}
```

**useAccessStore 状态**：

```typescript
interface AccessState {
  accessCodes: string[];           // 权限码列表
  accessMenus: MenuRecordRaw[];    // 可访问的菜单列表
  accessRoutes: RouteRecordRaw[];  // 可访问的路由列表
  accessToken: string | null;      // 登录 accessToken
  isAccessChecked: boolean;        // 是否已检查权限
  refreshToken: string | null;     // 刷新 token
}
```

**useDictStore 设计**：

```typescript
// store/dict.ts
export const useDictStore = defineStore('app-dict', () => {
  // 字典缓存 Map
  const dictOptionsMap = reactive(new Map<string, DictOption[]>());
  
  // 请求状态缓存
  const dictRequestCache = reactive(new Map<string, Promise<DictData[]>>());

  // 获取字典 Options
  function getDictOptions(dictName: string): DictOption[] {
    if (!dictOptionsMap.has(dictName)) {
      dictOptionsMap.set(dictName, []);
    }
    return dictOptionsMap.get(dictName)!;
  }

  // 清空缓存
  function resetCache() {
    dictRequestCache.clear();
    dictOptionsMap.clear();
  }

  return { getDictOptions, setDictInfo, resetCache };
});
```

### 3.4 组件适配器

**设计理念**：

为了统一不同组件库的使用方式，项目实现了组件适配器模式。

**核心文件**：`adapter/component/index.ts`

```typescript
export type ComponentType = 
  | 'Select'
  | 'Input'
  | 'DatePicker'
  | 'Upload'
  | 'TreeSelect'
  | 'InputNumber'
  | 'Switch'
  | 'Checkbox'
  | 'Radio'
  // ... 其他组件

// 组件映射
const components: Partial<Record<ComponentType, Component>> = {
  Select: withDefaultPlaceholder(Select, 'select'),
  Input: withDefaultPlaceholder(Input, 'input'),
  DatePicker: withDefaultPlaceholder(DatePicker, 'select'),
  // ...
}

// 获取组件
export function getComponent(component: ComponentType) {
  return components[component];
}
```

**优势**：

- 统一组件使用方式
- 方便切换组件库
- 自动处理 placeholder 等通用属性

### 4.3 数据加密流程

项目使用 **RSA + AES** 混合加密机制：

- **RSA**：非对称加密，用于加密 AES 密钥
- **AES**：对称加密，用于加密请求/响应数据

**请求加密流程**：

```
1. 生成随机 AES 密钥 (32 位随机字符串)
       ↓
2. 使用 RSA 公钥加密 AES 密钥
       ↓
3. 将加密后的 AES 密钥放入请求头 encrypt-key
       ↓
4. 使用 AES 密钥加密请求体数据
       ↓
5. 发送加密请求
```

**响应解密流程**：

```
1. 从响应头获取 encrypt-key (RSA 加密的 AES 密钥)
       ↓
2. 使用 RSA 私钥解密得到 AES 密钥
       ↓
3. 使用 AES 密钥解密响应数据
```

**核心代码**：

```typescript
// utils/http/index.ts - beforeRequest
if (enableEncrypt && encrypt && ['POST', 'PUT'].includes(request.type)) {
  // 1. 生成随机 AES 密钥
  const key = randomStr(32);
  const keyWithBase64 = encodeBase64(key);
  
  // 2. RSA 加密 AES 密钥
  config.headers['encrypt-key'] = asymmetricEncryption.encrypt(keyWithBase64);
  
  // 3. AES 加密请求数据
  request.data = symmetricEncryption.encrypt(JSON.stringify(data), key);
}
```
