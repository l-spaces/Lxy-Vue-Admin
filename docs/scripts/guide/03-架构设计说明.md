# 架构设计说明

## 简介

本项目采用 Monorepo。应用层、组合能力层、核心能力层、工具层分工明确，`apps/web-antd` 负责最终组装。

## 适用范围

- 功能设计前的模块落点判断
- 共享能力抽取与依赖边界确认

## 对应源码目录或关键文件

- `apps/web-antd/src`
- `packages/@core/*`
- `packages/effects/*`
- `packages/{constants,icons,locales,preferences,stores,styles,types,utils}`
- `internal/*`
- `docs/.vitepress/config.mts`

## 核心机制或功能说明

### 分层结构

- 应用层：`apps/web-antd/src`
- 组合能力层：`packages/effects/*`
- 核心能力层：`packages/@core/*`
- 基础工具层：`packages/constants`、`packages/types`、`packages/utils` 等
- 工程工具层：`internal/*`

### 主应用结构（`apps/web-antd/src`）

- `adapter`：表单与组件适配
- `api`：接口组织
- `components`：应用层业务组件
- `layouts`：布局接入层
- `locales`：应用语言包装
- `router`：路由与守卫
- `store`：应用层 Pinia Store
- `utils`：HTTP、消息、上下文等工具
- `views`：页面视图

### packages 职责归类（当前源码）

- UI 组件/组件能力：
  - `@vben/common-ui`
  - `@vben-core/form-ui`
  - `@vben-core/layout-ui`
  - `@vben-core/menu-ui`
  - `@vben-core/popup-ui`
  - `@vben-core/shadcn-ui`
  - `@vben-core/tabs-ui`
- 布局能力：`@vben/layouts`
- 表单能力：`@vben-core/form-ui`
- 菜单/标签页/弹窗：`@vben-core/menu-ui`、`@vben-core/tabs-ui`、`@vben-core/popup-ui`
- 插件能力：`@vben/plugins`
- 请求能力：`@vben/request`
- 权限能力：`@vben/access`
- 工具与基础：`@vben/utils`、`@vben/constants`、`@vben/locales`、`@vben/preferences`、`@vben/stores`、`@vben/types`、`@vben/styles`、`@vben/icons`

### 文档站点组织

- 站点根：`docs`
- 文档主目录：`docs/scripts`
- 站点配置：`docs/.vitepress/config.mts`

## 使用方式、扩展方式或注意事项

- 新增通用能力时，优先判断应放入 `packages` 还是仅应用层。
- `components` 文档只记录组件与组件能力，不把纯工具包写进组件体系。
- 变更目录结构后必须同步 `docs/.vitepress/config.mts` 与 `reference` 索引。
