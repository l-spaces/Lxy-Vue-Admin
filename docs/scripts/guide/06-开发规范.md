# 开发规范

## 简介

本规范基于当前仓库实现，不定义脱离源码的流程。

## 适用范围

- `apps/web-antd` 业务开发
- `packages` 共享能力开发
- 文档同步维护

## 对应源码目录或关键文件

- `apps/web-antd/package.json`
- 根 `package.json`
- `apps/web-antd/src/adapter/*`
- `apps/web-antd/src/api/*`
- `apps/web-antd/src/locales/langs/*`
- `internal/lint-configs/*`

## 核心机制或功能说明

### 代码与目录约定

- 路径别名：`#/` -> `apps/web-antd/src/*`
- API 分层：按业务模块放在 `src/api/<domain>/<module>`
- 页面逻辑优先使用 `<script setup lang="ts">`
- 表单优先走 `useVbenForm + 适配器组件`

### 接口与类型约定

- API 文件：`index.ts`
- 模块类型：`model.d.ts`（存在即优先复用）
- 请求统一使用 `alovaInstance`

### 国际化约定

- 用户可见文案使用 `$t(...)`
- 应用语言包维护在 `apps/web-antd/src/locales/langs`

### 常用质量命令

```bash
pnpm lint
pnpm check:type
pnpm check
```

### 提交规范

- 格式：`<type>(<scope>): <subject>`
- 常用 type：`feat`、`fix`、`docs`、`refactor`、`chore`、`test`
- 交互式提交：`pnpm commit`

## 使用方式、扩展方式或注意事项

- 新增功能时先补类型再写页面逻辑，减少后续接口改动成本。
- 若修改共享包导出，需要同步应用引用与文档入口。
- 文档改动时同步检查 `docs/.vitepress/config.mts` 和 `reference/文档目录.md`。
