# 项目概述

## 简介

`Lxy-Vue-Admin` 是基于 `Vben Admin 5.7.0`、`Vue 3.5`、`Vite 8` 和 `antdv-next` 的企业后台前端项目，采用 `pnpm workspace + Turbo` 的 Monorepo 结构。

## 适用范围

- 了解项目边界和主要技术选型
- 快速定位主应用、共享包和内部工具目录

## 对应源码目录或关键文件

- 根配置：`package.json`、`pnpm-workspace.yaml`、`turbo.json`
- 主应用：`apps/web-antd`
- 共享包：`packages/*`
- 内部工具：`internal/*`

## 核心机制或功能说明

### 技术栈（以当前源码依赖为准）

- 核心：`vue@^3.5.30`、`vite@^8.0.1`、`typescript@^5.9.3`
- 路由与状态：`vue-router@^5.0.4`、`pinia@^3.0.4`
- UI：`antdv-next@^1.1.7`
- 请求：`alova@^3.4.1` + `axios@^1.13.6`
- 工程：`pnpm@10.32.1`、`turbo@^2.8.20`

### Monorepo 主体目录

- `apps/web-antd`：业务应用集成层
- `packages/@core/*`：基础与 UI 套件核心能力
- `packages/effects/*`：组合能力（布局、通用 UI、请求、插件、权限）
- `packages/{constants,icons,locales,preferences,stores,styles,types,utils}`：通用基础包
- `internal/*`：lint、tsconfig、vite-config、tailwind-config 等内部工具
- `docs`：VitePress 站点

## 使用方式、扩展方式或注意事项

- 新业务页面优先落在 `apps/web-antd/src/views`。
- 共享能力优先复用 `packages` 现有模块，避免在应用层重复封装。
- 文档阅读建议先看 [架构设计说明](./03-架构设计说明.md) 与 [核心模块功能详解](./04-核心模块功能详解.md)。
- API 说明入口在 [API 文档](../api/README.md)。
