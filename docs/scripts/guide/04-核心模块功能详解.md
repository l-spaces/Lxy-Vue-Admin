# 核心模块功能详解

## 简介

本文档聚焦主应用运行主链路：启动、路由、权限、状态、请求、国际化和偏好配置。

## 适用范围

- 新功能接入前理解框架主流程
- 调试登录、路由、菜单、请求问题

## 对应源码目录或关键文件

- `apps/web-antd/src/main.ts`
- `apps/web-antd/src/bootstrap.ts`
- `apps/web-antd/src/router/{index,guard,access}.ts`
- `apps/web-antd/src/store/*`
- `apps/web-antd/src/utils/http/*`
- `apps/web-antd/src/locales/index.ts`
- `apps/web-antd/src/preferences.ts`
- `apps/web-antd/src/adapter/{form,component}/`

## 核心机制或功能说明

### 启动流程

`main.ts`：`initPreferences()` -> 动态导入 `bootstrap()` -> `unmountGlobalLoading()`

`bootstrap.ts` 关键步骤：

1. 初始化组件适配器 `initComponentAdapter`
2. 初始化表单适配 `initSetupVbenForm`
3. 设置弹窗默认参数 `setDefaultModalProps`
4. 注册全局组件和 `v-loading`
5. 初始化 i18n 与 Pinia
6. 注册权限指令 `v-access`
7. 初始化 `tippy`
8. 安装路由与守卫
9. 安装 Motion 插件
10. 监听路由动态更新标题

### 路由与权限

- 路由实例：`router/index.ts`
- 核心路由：`router/routes/core.ts`
- 守卫：`router/guard.ts`
  - `setupCommonGuard`：进度条和已加载路径
  - `setupAccessGuard`：token 检查、动态路由生成
- 后端菜单转换：`router/access.ts`
  - `Layout -> BasicLayout`
  - `ParentView -> ''`
  - `InnerLink + link -> IFrameView`
  - 支持 `RootMenu`、外链、内嵌和 query 处理

### 状态管理

- 应用层：`apps/web-antd/src/store`
  - `useAuthStore`：登录、登出、用户信息
  - `useDictStore`：字典缓存和请求去重
  - `useNotifyStore`：SSE 通知
  - `useTenantStore`：租户列表和选中态
- 框架层：`@vben/stores`

### HTTP 封装

- 入口：`utils/http/index.ts`
- 请求库：`alova + axios adapter`
- 默认能力：
  - token、语言头、`ClientID`
  - GET/DELETE 参数序列化 `arrayFormat: repeat`
  - 业务码 200 成功分支
  - 401 统一登出
  - 可选请求/响应加解密（RSA + AES）
  - `getWithMsg/postWithMsg/putWithMsg/deleteWithMsg`

### 国际化

- 入口：`locales/index.ts`
- 应用语言包：`locales/langs/{zh-CN,en-US}`
- 同步三方语言：`antdv-next locale` 与 `dayjs.locale`

### 偏好配置

- 文件：`preferences.ts`
- 当前覆盖重点：
  - `app.accessMode = 'mixed'`
  - `app.enableRefreshToken = false`
  - `footer.enable = false`
  - `tabbar.persist = false`
  - 主题色与圆角

## 使用方式、扩展方式或注意事项

- 若新增表单组件，需同步更新 `adapter/component/index.ts` 与 `adapter/form.ts`。
- 若后端菜单字段变更，需先修改 `router/access.ts` 的映射规则。
- 若开启加密，建议先以 `POST/PUT + encrypt: true` 小范围验证。
