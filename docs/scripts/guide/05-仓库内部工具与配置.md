# 仓库内部工具与配置

## 简介

`internal` 目录保存仓库级私有包和共享配置。它们不面向业务功能，而是为 monorepo 的 lint、构建、TypeScript 类型、样式和脚本流程提供基础能力。当前说明基于 `internal/README.md` 与各 package 源码整理，具体以源码导出为准。

## 适用范围

- 调整 lint 规则或提交规范
- 修改 Vite 构建与插件行为
- 统一 TypeScript / Tailwind / Node 工具的复用方式
- 理解 `apps/web-antd` 与 `packages` 对内部基础设施的依赖

## 对应源码目录或关键文件

- `internal/README.md`
- `internal/lint-configs/*`
- `internal/node-utils/src/*`
- `internal/vite-config/src/*`
- `internal/tsconfig/*.json`
- `internal/tailwind-config/src/*`

## 核心结构

### lint-configs

| 包名 | 目录 | 作用 |
| --- | --- | --- |
| `@vben/commitlint-config` | `internal/lint-configs/commitlint-config` | 提交信息规范与 scope 校验 |
| `@vben/eslint-config` | `internal/lint-configs/eslint-config` | ESLint Flat Config 入口，组合 Vue、TypeScript、JSON、YAML、pnpm 等规则 |
| `@vben/oxlint-config` | `internal/lint-configs/oxlint-config` | Oxlint 快速检查配置，作为 ESLint 的补充 |
| `@vben/oxfmt-config` | `internal/lint-configs/oxfmt-config` | 格式化配置封装 |
| `@vben/stylelint-config` | `internal/lint-configs/stylelint-config` | CSS / SCSS 检查配置 |

### node-utils

`@vben/node-utils` 提供仓库内脚本与配置包共用的 Node 工具。常见模块包括：

| 文件 | 作用 |
| --- | --- |
| `src/monorepo.ts` | 查找仓库根目录、读取包清单 |
| `src/fs.ts` | JSON 和文件读写 |
| `src/git.ts` | staged 文件与 Git 相关操作 |
| `src/hash.ts` | 内容哈希生成 |
| `src/date.ts` | `dayjs` 相关能力 |
| `src/path.ts` | 路径格式化 |
| `src/formatter.ts` | 文件格式化入口 |
| `src/spinner.ts` | CLI 加载态封装 |

### vite-config

`@vben/vite-config` 聚合了应用和库两类构建入口，核心导出包括 `defineConfig`、`defineApplicationConfig`、`defineLibraryConfig`、`loadApplicationPlugins` 和 `loadLibraryPlugins`。它内部组合了常见插件能力，例如国际化、打印信息、VXE Table 懒加载、启动页注入、压缩、PWA、HTML 处理、许可证检查、打包归档和构建可视化。

### tsconfig

`@vben/tsconfig` 通过 `internal/tsconfig/*.json` 提供共享模板，当前可见的配置有：

- `base.json`
- `node.json`
- `library.json`
- `web.json`
- `web-app.json`

### tailwind-config

`@vben/tailwind-config` 由 `src/index.ts` 和 `src/theme.css` 构成，作为 Tailwind 主题与公共样式变量的共享入口。

## 依赖关系

- `@vben/vite-config` 依赖 `@vben/node-utils`
- `@vben/eslint-config` 与 `@vben/oxlint-config` 互为补充，避免重复检查
- `@vben/tsconfig` 被应用和库型 package 继承
- `@vben/tailwind-config` 通常被应用的构建入口直接引入

## 使用与扩展注意事项

- `internal` 下的 package 都是仓库私有能力，`package.json` 中均为 `private: true`
- 修改导出时，先确认 `apps/web-antd`、`packages` 和脚本任务是否依赖现有 API
- 只调整源码和配置入口，不要把 `dist`、`node_modules` 等构建产物当作文档依据
- 需要更细粒度说明时，优先查看具体 package 的 `src/index.ts` 和相关实现文件
