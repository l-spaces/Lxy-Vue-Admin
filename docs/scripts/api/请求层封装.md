# 请求层封装

## 简介

当前项目请求层由 `alova + axios adapter` 实现，统一封装在 `apps/web-antd/src/utils/http/index.ts`。

## 适用范围

- 调整请求头、超时、加密逻辑
- 统一处理成功码、401、分页返回
- 排查接口报错弹窗与消息提示

## 对应源码目录或关键文件

- `apps/web-antd/src/utils/http/index.ts`
- `apps/web-antd/src/utils/http/checkStatus.ts`
- `apps/web-antd/src/utils/http/helper.ts`
- `packages/constants/src/core.ts`

## 核心机制或功能说明

### 实例基础配置

- `baseURL`：来源于 `useAppConfig(...).apiURL`
- `timeout`：`10_000`
- `cacheFor`：`null`（关闭 GET 缓存）
- `shareRequest`：`false`

### 请求前处理（`beforeRequest`）

- 合并默认 Axios 配置：
  - `successMessageMode: 'none'`
  - `errorMessageMode: 'message'`
  - `withToken: true`
  - `isReturnNativeResponse: false`
  - `isTransformResponse: true`
  - `encrypt: false`
- 注入头信息：
  - `Authorization: Bearer <token>`
  - `Accept-Language` / `Content-Language`（`zh_CN` 或 `en_US`）
  - `ClientID`
- GET/DELETE 参数序列化：
  - 默认使用 `stringify(params, { arrayFormat: 'repeat' })`
- 可选加密链路（仅 POST/PUT 且 `encrypt: true`）：
  - 生成随机 AES key
  - `encrypt-key` 头写入 RSA 加密后的 key
  - 请求体 AES 加密
  - 强制 `Content-Type: application/json;charset=UTF-8`

### 响应处理（`responded.onSuccess`）

- 若响应头存在 `encrypt-key`，先执行 RSA + AES 解密
- 按 `BUSINESS_SUCCESS_CODE = 200` 判断业务成功
- 成功时返回值规则：
  - 有 `data` 字段：直接返回 `data`
  - 无 `data` 字段：返回 `{ rows, total, ... }` 等剩余字段
- `UNAUTHORIZED_CODE = 401` 时执行统一登出 `handleUnauthorizedLogout()`
- 支持 `isReturnNativeResponse`、`isTransformResponse` 控制返回形态

### 错误处理（`responded.onError`）

- 网络超时与网络错误优先处理
- HTTP 状态码交由 `checkStatus` 映射提示：
  - `400/401/403/404/408/default`
- 登录超时流程由 `helper.ts` 的 `handleUnauthorizedLogout()` 管控

### 扩展请求方法

实例额外提供：

- `getWithMsg`
- `postWithMsg`
- `putWithMsg`
- `deleteWithMsg`

这些方法会在成功时统一触发成功消息提示。

## 使用方式、扩展方式或注意事项

- 需要请求加密时，在单个请求里显式传 `encrypt: true`。
- 分页接口建议维持后端 `{ code, msg, rows, total }` 结构，避免重复转换。
- 若出现 401 循环登出，优先检查后端 `logout` 接口返回是否异常。
