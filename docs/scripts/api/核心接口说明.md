# 核心接口说明

## 简介

本页按“认证与会话、菜单与用户、系统管理”整理当前项目核心接口，所有结论基于现有 `index.ts` 与类型定义。

## 适用范围

- 登录与鉴权链路联调
- 菜单、用户、角色等基础模块开发
- 快速确认分页与错误码返回格式

## 对应源码目录或关键文件

- `apps/web-antd/src/api/core/{auth,menu,user}.ts`
- `apps/web-antd/src/api/system/{user,role,menu,dict}`
- `apps/web-antd/src/api/common.d.ts`
- `apps/web-antd/src/utils/http/index.ts`

## 核心机制或功能说明

### 认证与会话（`api/core/auth.ts`）

- `loginApi(data)`：`POST /auth/login`，默认附带 `clientId`，并开启 `encrypt: true`
- `doLogout()`：`POST /auth/logout`
- `tenantList()`：`GET /auth/tenant/list`
- `authBinding(source, tenantId)`：第三方绑定地址
- `authCallback(data)`：`POST /auth/social/callback`

关键类型：

- `AuthApi.LoginParams`（`SimpleLoginParams | OAuthLoginParams`）
- `AuthApi.LoginResult`（`access_token/client_id/expire_in`）
- `TenantResp`（`tenantEnabled`、`voList`）

### 菜单与用户基础信息（`api/core/menu.ts`、`api/core/user.ts`）

- `getAllMenusApi()`：`GET /system/menu/getRouters`
- `getUserInfoApi()`：`GET /system/user/getInfo`

关键类型：

- `Menu` / `MenuMeta`
- `UserInfoResp`
- `User` / `Role`

### 系统管理高频模块

- 用户（`api/system/user/index.ts`）
  - `userList`、`findUserInfo`、`userAdd`、`userUpdate`、`userRemove`
  - `userResetPassword` 使用 `encrypt: true`
- 角色（`api/system/role/index.ts`）
  - `roleList`、`roleInfo`、`roleAdd`、`roleUpdate`、`roleRemove`
  - `roleDataScope`、角色授权相关接口
- 菜单（`api/system/menu/index.ts`）
  - `menuList`、`menuInfo`、`menuAdd`、`menuUpdate`、`menuRemove`
  - `menuTreeSelect`、`roleMenuTreeSelect`
- 字典（`api/system/dict/*`）
  - `dictDataInfo`、`dictDataList`
  - `dictTypeList`、`dictTypeInfo`

### 分页与返回格式

- 通用分页类型：`PageResult<T> = { rows: T[]; total: number }`
- 请求层成功码：`BUSINESS_SUCCESS_CODE = 200`
- 未授权码：`UNAUTHORIZED_CODE = 401`
- 成功返回时：
  - 有 `data` 字段返回 `data`
  - 无 `data`（常见分页）返回 `{ rows, total, ... }`

### 错误处理与提示

- HTTP 错误由 `checkStatus` 统一映射提示
- 401 触发 `handleUnauthorizedLogout()` 统一登出流程
- 可选 `getWithMsg/postWithMsg/putWithMsg/deleteWithMsg` 触发成功提示

## 使用方式、扩展方式或注意事项

- 业务代码引用核心接口可直接 `from '#/api'`，系统模块需按目录路径导入。
- 设计新接口时，优先复用 `PageQuery`、`PageResult`、`ID/IDS` 等通用类型。
- 发现接口文档与实现不一致时，以 `apps/web-antd/src/api` 与 `utils/http` 实际代码为准。
