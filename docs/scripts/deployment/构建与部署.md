# 构建与部署

## 简介

本页聚焦当前仓库已经落地的构建与部署路径：主应用、Docker 镜像、文档站点。

## 适用范围

- 发布 `web-antd`
- 生成 Docker 镜像
- 发布 VitePress 文档站点

## 对应源码目录或关键文件

- 根 `package.json`
- `apps/web-antd/package.json`
- `scripts/deploy/{Dockerfile,build-local-docker-image.sh}`
- `.github/workflows/deploy.yml`

## 核心机制或功能说明

### 主应用构建

根目录命令：

```bash
pnpm build:antd
pnpm build:antd:test
pnpm build:analyze
```

子包实际执行：

- `@vben/web-antd` 的 `build:prod` / `build:test` / `build:analyze`

### Docker 构建（仓库现有）

- Dockerfile：`scripts/deploy/Dockerfile`
  - Stage1：Node 22 + pnpm 构建 `@vben/web-antd`
  - Stage2：Nginx 承载 `apps/web-antd/dist`
- 本地脚本：`scripts/deploy/build-local-docker-image.sh`

### 文档站点发布

`.github/workflows/deploy.yml` 当前流程：

1. `pnpm install`
2. `pnpm docs:build`
3. 上传 `docs/.vitepress/dist`
4. 部署到 GitHub Pages

触发条件当前为：

- push 到 `master` 且命中 `docs/**` 或工作流文件变更
- 手动触发 `workflow_dispatch`

## 使用方式、扩展方式或注意事项

- 修改构建脚本后，需同步验证根命令与子包命令是否一致。
- Docker 与 Nginx 端口、路径必须和镜像内文件布局对齐。
- 文档新增后，发布前应先本地执行 `pnpm docs:build` 确认无死链或导航缺失。
