# 环境变量与代理

## 简介

本页说明部署时最容易出错的两类配置：环境变量与反向代理路径。

## 适用范围

- 生产部署配置
- 代理路径联调
- SSE/WebSocket 反向代理检查

## 对应源码目录或关键文件

- `apps/web-antd/.env.production`
- `apps/web-antd/.env.test`
- `packages/effects/hooks/src/use-app-config.ts`
- `apps/web-antd/vite.config.ts`
- `scripts/deploy/nginx.conf`

## 核心机制或功能说明

### 环境变量解析

`useAppConfig` 会读取：

- `VITE_GLOB_API_URL`
- `VITE_GLOB_APP_CLIENT_ID`
- `VITE_GLOB_ENABLE_ENCRYPT`
- `VITE_GLOB_RSA_PUBLIC_KEY`
- `VITE_GLOB_RSA_PRIVATE_KEY`
- `VITE_GLOB_SSE_ENABLE`
- `VITE_GLOB_WEBSOCKET_ENABLE`

生产环境下这些值来自 `window._VBEN_ADMIN_PRO_APP_CONF_`。

### 开发代理

`apps/web-antd/vite.config.ts` 当前配置：

- `/api -> http://localhost:8080`
- `rewrite(/^\/api/, '')`
- `ws: true`

### Nginx 生产代理

`scripts/deploy/nginx.conf` 当前配置重点：

- 前端路由回退：`try_files $uri $uri/ /index.html`
- 后端代理：`/prod-api/` -> `upstream server`
- SSE/WebSocket 支持：
  - `proxy_http_version 1.1`
  - `Upgrade`/`Connection` 头透传
  - `proxy_buffering off`

## 使用方式、扩展方式或注意事项

- 如果前端 `base` 为 `/vben`，Nginx 的静态目录与回退路径要保持一致。
- 若启用 SSE/WebSocket，反向代理必须保留升级头和长连接超时设置。
- `VITE_GLOB_API_URL` 与反向代理路径应统一设计，避免“前端直连 + 代理”混用。
