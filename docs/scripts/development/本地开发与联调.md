# 本地开发与联调

## 简介

本页说明 `web-antd` 在本地联调时的最小流程，包含环境变量和代理路径的确认点。

## 适用范围

- 本地首次启动
- 前后端联调
- 请求路径异常排查

## 对应源码目录或关键文件

- `apps/web-antd/.env*`
- `apps/web-antd/vite.config.ts`
- `packages/effects/hooks/src/use-app-config.ts`
- `apps/web-antd/src/utils/http/index.ts`

## 核心机制或功能说明

### 启动步骤

1. 安装依赖：`pnpm install`
2. 启动主应用：`pnpm dev:antd`
3. 默认访问：`http://localhost:5666`（来自 `.env.development` 的 `VITE_PORT`）

### 联调关键配置

- `VITE_GLOB_API_URL`：
  - 直接写后端地址时，请求走该地址
  - 写相对路径时通常配合 Vite 代理
- Vite 代理（`vite.config.ts`）：
  - `/api -> http://localhost:8080`
  - `rewrite: ^/api -> ''`
  - `ws: true`

### 请求头与语言

请求层会自动注入：

- `Authorization`
- `ClientID`
- `Accept-Language`
- `Content-Language`

## 使用方式、扩展方式或注意事项

- 若接口 404，先确认是否走了 `/api` 代理，还是直接请求了 `VITE_GLOB_API_URL`。
- 若后端要求加密，请确认 `VITE_GLOB_ENABLE_ENCRYPT` 和密钥配置是否匹配。
- 若切换了 `preferences` 或关键环境变量，建议清理浏览器缓存后再验证。
