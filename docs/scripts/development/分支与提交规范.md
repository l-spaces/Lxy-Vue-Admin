# 分支与提交规范

## 简介

本页区分“仓库已强制约束”和“团队约定”，避免把建议误写成硬规则。

## 适用范围

- 代码提交流程
- commit message 规范
- pre-commit hook 行为确认

## 对应源码目录或关键文件

- `lefthook.yml`
- `internal/lint-configs/commitlint-config/index.mjs`
- `package.json`
- `AGENTS.md`

## 核心机制或功能说明

### 已强制执行的约束

- `commit-msg` 钩子执行：`pnpm exec commitlint --edit $1`
- `type-enum` 限制：
  - `feat`
  - `fix`
  - `perf`
  - `style`
  - `docs`
  - `test`
  - `refactor`
  - `build`
  - `ci`
  - `chore`
  - `revert`
  - `types`
  - `release`
  - `update`
- `header-max-length`：108
- `subject-empty`：禁止为空
- `scope`：若填写，需命中允许列表（workspace 包名或 `project/style/lint/ci/dev/deploy/other`）

### pre-commit 钩子行为

`lefthook.yml` 当前会在暂存文件上执行：

- `oxfmt`
- `oxlint --fix`
- `eslint --fix`
- `stylelint --fix`

并按文件类型分流执行对应任务。

### 分支命名约定

- 仓库脚本未发现“分支命名”硬校验规则。
- `AGENTS.md` 中给出了团队约定示例：
  - `master`
  - `develop`
  - `feature/*`
  - `bugfix/*`
  - `release/*`
  - `hotfix/*`

## 使用方式、扩展方式或注意事项

- 建议使用 `pnpm commit`（`czg`）降低提交格式出错概率。
- 调整 commit 规则时，应同时修改 `commitlint-config` 与团队文档。
- 若新增分支策略规则，需要额外引入 CI 或 hook 校验，当前仓库未内置。
