# 登录页面架构分析

## 简介

登录页采用“布局层 + 通用 UI 层 + 应用业务层”分层组合，既复用通用能力，又保留租户、验证码、第三方登录等业务差异。

## 适用范围

- 登录页问题排查
- 新增登录方式（短信、社交等）扩展
- 分层职责确认

## 对应源码目录或关键文件

- `apps/web-antd/src/views/_core/authentication/login.vue`
- `apps/web-antd/src/layouts/auth.vue`
- `packages/effects/layouts/src/authentication/*`
- `packages/effects/common-ui/src/ui/authentication/*`
- `apps/web-antd/src/store/auth.ts`
- `apps/web-antd/src/api/core/{auth,captcha}.ts`

## 核心机制或功能说明

### 组件分层

| 层级 | 主要文件 | 职责 |
| --- | --- | --- |
| 布局层 | `packages/effects/layouts/src/authentication/authentication.vue` | logo、工具栏、认证页整体布局 |
| 通用登录 UI 层 | `packages/effects/common-ui/src/ui/authentication/*` | 登录表单渲染、交互容器、基础提交逻辑 |
| 应用业务层 | `apps/web-antd/src/views/_core/authentication/login.vue` | 租户、验证码、OAuth 等项目定制 |

### 登录主流程

1. 页面加载时请求租户列表与验证码（按配置决定显示）
2. 用户提交表单后调用 `useAuthStore().authLogin(...)`
3. `authLogin` 内部：
   - 调 `loginApi`
   - 写入 token（`useAccessStore`）
   - 调 `fetchUserInfo`
   - 设置权限码并跳转首页

### 关键扩展点

- 表单组件与字段通过 Schema 生成（由通用登录组件承载）
- 第三方登录通过业务层 slot/组件插入
- 租户逻辑由 `tenantList()` + `useTenantStore` 驱动

## 使用方式、扩展方式或注意事项

- 新增登录方式优先扩展应用层参数组装，避免直接改通用层基础协议。
- 登录流程问题优先检查：
  - `api/core/auth.ts`
  - `store/auth.ts`
  - 路由守卫 `router/guard.ts`
- 若出现“登录成功但菜单不加载”，重点检查 token、`fetchUserInfo` 与 `generateAccess` 调用链。
