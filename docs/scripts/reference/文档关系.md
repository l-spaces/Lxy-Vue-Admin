# 文档关系

## 简介

本页描述各分类文档间的引用方向，避免信息重复和职责重叠。

## 适用范围

- 新增文档时确定落点
- 调整目录结构时检查影响面

## 对应源码目录或关键文件

- `docs/scripts/*`

## 核心机制或功能说明

### 主关系链路

1. `guide` 定义项目全局背景与运行机制
2. `api`、`components` 承接专题能力说明
3. `development`、`deployment`、`maintenance` 承接执行流程
4. `analysis` 记录结构性观察结论
5. `reference` 负责聚合索引与导航回查

### 交叉引用约定

- `guide` 可以链接到 `api/components/development`，不重复写细节
- `api/components` 中如需环境前提，回链 `guide/02-环境配置指南`
- `maintenance` 处理故障场景时，回链 `api/请求层封装` 或 `guide/04-核心模块功能详解`

### 单点维护约束

- 目录结构清单只在 `reference/文档目录.md` 维护
- 站点导航只在 `docs/.vitepress/config.mts` 维护
- 分类入口只在各目录 `README.md` 维护

## 使用方式、扩展方式或注意事项

- 发现重复内容时，保留信息更完整的一处，其他文档改为链接引用。
- 变更一处入口时，至少联动检查 `README`、`reference` 与 `vitepress` 配置三处。
