## 文档维护指南

### 新增文档

当需要在项目中添加新的 Markdown 文档时，请遵循以下步骤：

1. **确定文档位置**: 根据文档类型选择合适的目录
   - 项目级文档 → 项目根目录
   - 开发指南 → `docs/` 目录
   - 包文档 → 对应的 `packages/` 子目录
   - 应用文档 → 对应的 `apps/` 子目录

2. **命名规范**: 
   - 使用清晰、描述性的文件名
   - 中文文档使用中文命名
   - 英文文档使用小写字母和连字符

3. **更新目录**: 在本文档中添加新文档的链接和说明

### 文档分类说明

| 目录 | 用途 | 示例 |
|------|------|------|
| `/` | 项目级核心文档 | README.md, CHANGELOG.md |
| `/docs/` | 开发文档和指南 | 开发文档.md, 架构分析 |
| `/packages/` | 包的说明文档 | 各包的 README.md |
| `/apps/` | 应用相关文档 | 视图说明、组件文档 |
| `/scripts/` | 脚本工具文档 | 代码生成模板说明 |
| `/internal/` | 内部模块文档 | Vite 配置说明 |

### 链接格式规范

本文档中的所有链接均采用相对路径格式：

```markdown
[文档名称](./相对路径/文件名.md)
```

**注意事项**:
- 使用 `./` 开头表示相对路径
- 路径中的空格无需转义
- 确保文件名大小写正确
- 定期检查链接有效性
