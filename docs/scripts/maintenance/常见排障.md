# 常见排障

## 简介

本页聚焦当前代码中最常见的故障点，并给出对应源码定位入口。

## 适用范围

- 登录与权限异常
- 接口请求异常
- 页面数据/字典异常

## 对应源码目录或关键文件

- `apps/web-antd/src/router/guard.ts`
- `apps/web-antd/src/utils/http/index.ts`
- `apps/web-antd/src/utils/http/helper.ts`
- `apps/web-antd/src/store/{auth,dict,tenant,notify}.ts`
- `apps/web-antd/src/utils/message.ts`

## 核心机制或功能说明

### 登录后跳转异常

优先检查：

- `useAuthStore().authLogin` 是否成功设置 token
- `router/guard.ts` 中 `accessStore.isAccessChecked` 状态
- `generateAccess` 是否拉取到菜单并成功注册路由

### 401 循环或重复登出

优先检查：

- `handleUnauthorizedLogout()` 的防重入逻辑
- 后端 `logout` 接口是否错误返回 401（源码对此有异常分支）

### 列表接口分页显示异常

优先检查：

- 后端是否返回 `rows + total`
- 请求层 `onSuccess` 是否走到“无 data 字段返回 other”分支

### 字典重复请求或数据不刷新

优先检查：

- `useDictStore` 的 `dictRequestCache` 与 `dictOptionsMap`
- 切换用户/租户后是否触发 `dictStore.resetCache()`

### SSE/WebSocket 无消息

优先检查：

- `VITE_GLOB_SSE_ENABLE` / `VITE_GLOB_WEBSOCKET_ENABLE`
- `utils/message.ts` 中连接地址拼装
- 代理层是否透传升级头与关闭缓冲

## 使用方式、扩展方式或注意事项

- 建议先查浏览器 Network 与 Console，再进入源码定位。
- 同类错误优先修复统一封装层（HTTP、router guard、store），避免页面级重复补丁。
- 文档中未覆盖的异常场景，按“入口层 -> 共享层 -> 页面层”顺序追踪。
