# 依赖升级与日常维护

## 简介

本页基于当前仓库命令与工具链，整理依赖升级和日常维护的可执行流程。

## 适用范围

- 周期性依赖升级
- 发版前质量检查
- 日常健康巡检

## 对应源码目录或关键文件

- `package.json`
- `lefthook.yml`
- `oxfmt.config.ts`
- `oxlint.config.ts`
- `internal/lint-configs/*`

## 核心机制或功能说明

### 依赖升级命令

```bash
pnpm update:deps
pnpm install
```

如需全面验证：

```bash
pnpm lint
pnpm check:type
pnpm check
pnpm build:antd
pnpm docs:build
```

### 提交前自动检查

`lefthook.yml` 已配置 pre-commit：

- Markdown：`oxfmt`
- JS/TS/Vue：`oxfmt + oxlint + eslint + stylelint`
- commit-msg：`commitlint`

### 日常维护建议节奏

- 每周：
  - 跑一次 `pnpm check`
  - 检查关键页面联调
- 每次升级后：
  - 回归登录、动态路由、核心表格页、上传页
- 发版前：
  - 同步检查 `docs` 构建与文档入口可访问性

## 使用方式、扩展方式或注意事项

- 升级依赖时优先小步提交，便于回滚。
- 若升级涉及 `@vben/*` workspace 包，需关注跨包联动影响。
- 维护记录建议同步到变更日志，避免重复排障。
