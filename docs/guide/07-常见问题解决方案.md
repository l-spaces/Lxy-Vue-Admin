## 1. 环境配置问题

### 1.1 Node 版本不兼容

**问题描述**：

```
error: The engine "node" is incompatible with this module.
```

**解决方案**：

```bash
# 方案一：使用 nvm 切换 Node 版本
nvm install 22.18.0
nvm use 22.18.0

# 验证版本
node -v

# 方案二：使用 .nvmrc 文件
echo "22.18.0" > .nvmrc
nvm use
```

### 1.2 pnpm 依赖安装失败

**问题描述**：

```
ERR_PNPM_FETCH_*
```

**解决方案**：

```bash
# 方案一：清理缓存
pnpm store prune

# 方案二：删除 node_modules 重新安装
rm -rf node_modules pnpm-lock.yaml
pnpm install

# 方案三：使用国内镜像
pnpm config set registry https://registry.npmmirror.com

# 方案四：配置 .npmrc
# .npmrc
registry=https://registry.npmmirror.com
shamefully-hoist=true
strict-peer-dependencies=false
```

### 1.3 Python 环境问题

**问题描述**：

某些依赖需要 Python 环境编译。

**解决方案**：

```bash
# Windows
npm install -g windows-build-tools

# macOS
xcode-select --install

# Linux
sudo apt-get install python3 python3-pip
```

## 2. 开发常见问题

### 2.1 热更新不生效

**问题描述**：

修改代码后页面不自动刷新。

**解决方案**：

```typescript
// 方案一：检查 vite.config.ts 配置
export default defineConfig({
  server: {
    hmr: true, // 确保 HMR 开启
  },
});

// 方案二：清除浏览器缓存
// Chrome: F12 -> Network -> Disable cache

// 方案三：重启开发服务器
pnpm dev:antd

// 方案四：检查文件监听数量
// macOS/Linux
echo fs.inotify.max_user_watches=524288 | sudo tee -a /etc/sysctl.conf
sudo sysctl -p
```

### 2.2 TypeScript 类型错误

**问题描述**：

IDE 报类型错误但代码能运行。

**解决方案**：

```bash
# 方案一：重新生成类型
pnpm typecheck

# 方案二：重启 TypeScript 服务
# VS Code: Cmd+Shift+P -> TypeScript: Restart TS Server

# 方案三：检查 tsconfig.json 配置
{
  "compilerOptions": {
    "skipLibCheck": true,
    "noImplicitAny": false
  }
}

# 方案四：添加类型声明
// types/global.d.ts
declare module '*.vue' {
  import type { DefineComponent } from 'vue';
  const component: DefineComponent<{}, {}, any>;
  export default component;
}
```

### 2.3 跨域问题

**问题描述**：

```
Access to XMLHttpRequest at 'xxx' from origin 'xxx' has been blocked by CORS policy
```

**解决方案**：

```typescript
// 方案一：配置开发代理
// apps/web-antd/vite.config.ts
export default defineConfig({
  server: {
    proxy: {
      '/prod-api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/prod-api/, ''),
      },
    },
  },
});

// 方案二：后端配置 CORS
// Spring Boot
@CrossOrigin(origins = "*")

// 方案三：使用 nginx 反向代理
location /prod-api/ {
  proxy_pass http://backend:8080/;
}
```

### 2.4 组件样式不生效

**问题描述**：

修改组件样式后没有变化。

**解决方案**：

```vue
<!-- 方案一：使用 scoped 样式 -->
<style scoped>
.container {
  padding: 16px;
}
</style>

<!-- 方案二：使用深度选择器 -->
<style scoped>
:deep(.ant-input) {
  border-radius: 4px;
}
</style>

<!-- 方案三：使用全局样式 -->
<style>
.global-style {
  color: red;
}
</style>

<!-- 方案四：使用 CSS Modules -->
<style module>
.container {
  padding: 16px;
}
</style>
```

### 2.5 路由跳转失败

**问题描述**：

路由跳转后页面空白或 404。

**解决方案**：

```typescript
// 方案一：检查路由配置
const routes = [
  {
    path: '/user',
    name: 'User',
    component: () => import('#/views/system/user/index.vue'),
  },
];

// 方案二：检查路由守卫
router.beforeEach((to, from, next) => {
  // 确保调用 next()
  next();
});

// 方案三：检查动态路由注册
router.addRoute(route);

// 方案四：检查路由模式
// history 模式需要服务器配置
// nginx
location / {
  try_files $uri $uri/ /index.html;
}
```

## 3. 构建部署问题

### 3.1 打包后页面空白

**问题描述**：

部署后页面白屏，控制台报错。

**解决方案**：

```typescript
// 方案一：检查 base 路径配置
// .env.production
VITE_BASE=/vben/

// 方案二：检查路由模式
VITE_ROUTER_HISTORY=history  // 或 hash

// 方案三：检查静态资源路径
// vite.config.ts
export default defineConfig({
  base: '/vben/',
  build: {
    assetsDir: 'assets',
  },
});

// 方案四：检查服务器配置
// nginx
server {
  location /vben/ {
    alias /path/to/dist/;
    try_files $uri $uri/ /vben/index.html;
  }
}
```

### 3.2 静态资源 404

**问题描述**：

CSS/JS 文件加载失败。

**解决方案**：

```typescript
// 方案一：检查 publicPath
// vite.config.ts
export default defineConfig({
  base: './', // 相对路径
});

// 方案二：检查资源引用
// 使用绝对路径
import logo from '@/assets/logo.png';

// 方案三：检查 CDN 配置
// .env.production
VITE_CDN_URL=https://cdn.example.com
```

### 3.3 打包体积过大

**问题描述**：

打包后的文件体积过大，加载慢。

**解决方案**：

```typescript
// 方案一：开启代码分割
// vite.config.ts
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          'vendor': ['vue', 'vue-router', 'pinia'],
          'antd': ['antdv-next'],
          'echarts': ['echarts'],
        },
      },
    },
  },
});

// 方案二：开启压缩
// .env.production
VITE_COMPRESS=gzip

// 方案三：分析打包结果
pnpm build:analyze

// 方案四：按需引入组件
// 使用 unplugin-vue-components
```

### 3.4 环境变量不生效

**问题描述**：

打包后环境变量没有正确读取。

**解决方案**：

```typescript
// 方案一：确保变量以 VITE_ 开头
VITE_API_URL=/api

// 方案二：正确访问环境变量
const apiUrl = import.meta.env.VITE_API_URL;

// 方案三：检查 .env 文件位置
// apps/web-antd/.env.production

// 方案四：定义类型
// env.d.ts
interface ImportMetaEnv {
  readonly VITE_API_URL: string;
  readonly VITE_APP_TITLE: string;
}
```

## 5. 性能优化问题

### 5.1 页面加载慢

**问题描述**：

首次加载页面时间过长。

**解决方案**：

```typescript
// 方案一：路由懒加载
const routes = [
  {
    path: '/user',
    component: () => import('#/views/system/user/index.vue'),
  },
];

// 方案二：组件懒加载
const HeavyComponent = defineAsyncComponent(
  () => import('./HeavyComponent.vue')
);

// 方案三：开启预加载
// vite.config.ts
export default defineConfig({
  build: {
    modulePreload: {
      polyfill: true,
    },
  },
});

// 方案四：使用骨架屏
<Skeleton :loading="loading">
  <Content />
</Skeleton>
```

### 5.2 表格渲染慢

**问题描述**：

大数据量表格渲染卡顿。

**解决方案**：

```typescript
// 方案一：使用虚拟滚动
const gridOptions = {
  scrollY: {
    enabled: true,
    type: 'virtual',
    gt: 100,
  },
};

// 方案二：分页加载
const pagination = {
  pageSize: 20,
  showSizeChanger: true,
};

// 方案三：减少响应式数据
// 使用 shallowRef 代替 ref
const data = shallowRef<Row[]>([]);

// 方案四：冻结数据
// 使用 Object.freeze
const data = Object.freeze(rawData);
```

### 5.3 内存泄漏

**问题描述**：

页面使用一段时间后变慢。

**解决方案**：

```typescript
// 方案一：清理定时器
onUnmounted(() => {
  clearInterval(timer);
  clearTimeout(timeout);
});

// 方案二：移除事件监听
onUnmounted(() => {
  window.removeEventListener('resize', handleResize);
});

// 方案三：取消请求
const controller = new AbortController();
onUnmounted(() => {
  controller.abort();
});

// 方案四：清理 Store
onUnmounted(() => {
  store.$reset();
});
```

## 6. 调试技巧

### 6.1 Vue DevTools

**安装**：

- Chrome: [Vue.js devtools](https://chrome.google.com/webstore/detail/vuejs-devtools/nhdogjmejiglipccpnnnanhbledajbpd)
- Firefox: [Vue.js devtools](https://addons.mozilla.org/en-US/firefox/addon/vue-js-devtools/)

**使用**：

1. **Components 面板**：查看组件树和状态
2. **Pinia 面板**：查看 Store 状态
3. **Timeline 面板**：查看性能和时间线
4. **Router 面板**：查看路由信息

### 6.2 网络请求调试

**浏览器 DevTools**：

1. 打开 Network 面板
2. 筛选 XHR/Fetch 请求
3. 查看请求详情、响应数据

**Alova 调试**：

```typescript
// 开启调试日志
const alovaInstance = createAlova({
  responded: {
    onSuccess: (response) => {
      console.log('请求成功:', response);
      return response;
    },
    onError: (error) => {
      console.error('请求失败:', error);
      throw error;
    },
  },
});
```

### 6.3 性能分析

**Chrome Performance**：

1. 打开 DevTools -> Performance
2. 点击录制按钮
3. 执行操作
4. 停止录制并分析

**Vite 构建分析**：

```bash
# 分析打包结果
pnpm build:analyze

# 查看依赖关系
npx vite-bundle-visualizer
```

### 6.4 常用调试代码

```typescript
// 打印组件 props
watch(() => props, (newProps) => {
  console.log('Props changed:', newProps);
}, { deep: true });

// 打印路由变化
router.afterEach((to, from) => {
  console.log('Route changed:', from.path, '->', to.path);
});

// 打印 Store 变化
store.$subscribe((mutation, state) => {
  console.log('Store changed:', mutation.type, state);
});

// 性能计时
console.time('operation');
// ... 操作
console.timeEnd('operation');
```
