# 代码审查器

专业的代码审查技能，分析代码质量，识别潜在问题，提供可操作的改进建议。

## 触发条件

- 用户明确请求代码审查
- 提交或合并重要代码变更前
- 改进代码质量或重构时
- 完成功能开发后
- 用户询问代码最佳实践时

## 审查清单

### 1. 代码质量
- 代码可读性和可维护性
- 命名规范（变量、函数、类）
- 代码结构和组织
- DRY 原则遵循情况
- 单一职责原则

### 2. 潜在 Bug
- 空值/未定义检查
- 边界情况处理
- 错误处理完整性
- 竞态条件
- 内存泄漏

### 3. 安全问题
- 输入验证
- XSS/注入漏洞
- 敏感数据暴露
- 认证/授权检查
- 依赖项漏洞

### 4. 性能优化
- 不必要的计算
- 内存效率
- Async/await 使用
- 数据库查询优化
- 缓存机会

### 5. 最佳实践
- 框架特定规范
- 设计模式使用
- 代码文档
- 测试覆盖率
- 类型安全（TypeScript）

## 输出格式

```markdown
## 代码审查报告

### 总体评价
[代码质量简要总结]

### 发现的问题

#### 严重
- [必须修复的严重问题]

#### 警告
- [应该处理的重要问题]

#### 建议
- [改进建议]

### 优点
[代码做得好的地方]

### 改进建议
[可操作的改进建议]
```

## 使用示例

用户: "请审查我的认证模块"

技能将：
1. 分析认证代码
2. 检查安全漏洞
3. 审查错误处理
4. 提出改进建议
5. 提供结构化的审查报告
